# ADR-2026-06-inbound-only-gtm: Ship ask-the-board as one open-source BYOK engine with an inbound-only (pull) go-to-market -- free product, adoption first, monetize later via a capped hosted tier -- or run an outbound motion to seed demand?

- **Status:** Pre-registered (resolves 2026-09-24)
- **Anchored:** 2026-06-26T14:30:00
- **Resolution date:** 2026-09-24

## Context (stated prior)

I want them to chase me. The artifact is a git-committable board-minute -- developer-shaped and self-distributing -- so I am betting that a free, honest, zero-COGS engine plus one credible launch post is enough for inbound to start, and that outbound would cheapen it.

## Decision

Launch inbound-only: a single MIT-licensed BYOK engine, no outbound outreach, mass adoption ahead of monetization.

## Pre-registered prediction

> By 2026-09-24, with zero outbound outreach performed, the inbound-only launch will produce at least one unsolicited external pull signal -- defined as ANY of: (a) a GitHub issue or pull request opened by someone other than the author, (b) a genuine hosted-tier waitlist email to support@chu6a.dev, or (c) the repository crossing 25 stars.
